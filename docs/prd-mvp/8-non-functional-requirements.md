# 8. 非功能性需求

### 8.1 性能要求

| 指标              | 目标值  | 说明         |
| ----------------- | ------- | ------------ |
| API 响应时间      | < 100ms | 95% 的请求   |
| 页面首次加载      | < 2s    | 包含所有资源 |
| 视图切换          | < 200ms | 前端渲染     |
| Markdown 文件读取 | < 50ms  | 单个文件     |
| 数据库查询        | < 30ms  | 简单查询     |
| 支持 TODO 数量    | 10,000+ | 单机性能     |

### 8.2 可用性要求

- **浏览器兼容**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **屏幕分辨率**: 最小 1280x720
- **移动端**: 支持 iOS Safari, Chrome for Android

### 8.3 安全性要求

- **认证**: Cookie 使用 HttpOnly + Secure 标志
- **密码**: 硬编码密码建议使用环境变量配置
- **HTTPS**: 生产环境强制使用 HTTPS
- **CORS**: 限制允许的来源

### 8.4 可维护性要求

- **代码规范**:
  - C# 使用 .editorconfig 规范
  - TypeScript 使用 ESLint + Prettier
- **注释**: 关键业务逻辑需要注释
- **日志**: 使用结构化日志（Serilog）
- **错误处理**: 统一的异常处理中间件

### 8.5 可扩展性设计

**为后续功能预留扩展点**:

1. **扩展字段机制**:
   - YAML Front Matter 支持自定义字段
   - 数据库表设计时预留 `ExtendedData` JSON 字段

2. **关系运算**:
   - 数据库表预留 `Relations` 表（暂不使用）

3. **AI 功能**:
   - API 设计时预留 `/api/ai/*` 路由

4. **多用户**:
   - 数据库表预留 `UserId` 字段（MVP 固定为 `admin`）
